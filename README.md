# Discord Music Bot

Discord-бот на JavaScript, який потоково відтворює аудіо з YouTube у голосовому каналі, підтримує окрему чергу для кожного Discord-сервера та запускається локально або в Docker.

## Функції

- потокове відтворення YouTube без збереження аудіофайлів на диск;
- окремі черги, гучність і стан відтворення для кожного Discord-сервера;
- автоматичний запуск наступного треку;
- pause, resume, skip, stop і очищення черги;
- інтерактивна панель із Play/Pause, Stop, Skip і регулюванням гучності;
- швидка зміна гучності на ±10% або введення точного значення від 0 до 100;
- додавання нового YouTube URL через кнопку та Discord Modal;
- прогрес-бар поточного треку з автоматичним оновленням;
- безпечний запуск `yt-dlp` та `ffmpeg` через `spawn` без shell-команд;
- коректне завершення дочірніх процесів під час skip, stop і вимкнення бота;
- slash-команди Discord;
- Docker Compose із політикою автоматичного перезапуску.

## Вимоги

Для локального запуску:

- Node.js 22.12 або новіший;
- `yt-dlp` у `PATH`;
- `ffmpeg` у `PATH`;
- Discord Application із ботом.

На Windows залежності можна встановити так:

```powershell
python -m pip install -U --pre "yt-dlp[default]"
winget install --id Gyan.FFmpeg --exact
```

Після встановлення закрий і повторно відкрий термінал, щоб оновився `PATH`.

Для контейнерного запуску достатньо Docker із підтримкою Docker Compose.

## Налаштування Discord Application

1. Створи application у [Discord Developer Portal](https://discord.com/developers/applications).
2. На вкладці **Bot** створи бота й скопіюй token.
3. У **OAuth2 → URL Generator** обери scopes `bot` та `applications.commands`.
4. Додай дозволи бота:
   - View Channels;
   - Send Messages;
   - Connect;
   - Speak.
5. Відкрий згенероване посилання та запроси бота на сервер.
6. У Discord увімкни Developer Mode і скопіюй ID application та, за потреби, ID тестового сервера.

Правильне посилання для запрошення можна згенерувати автоматично:

```bash
npm run invite
```

Відкрий отримане посилання та вибери сервер. Воно встановлює саме серверну версію застосунку разом із ботом і потрібними дозволами.

Привілейовані Gateway Intents цьому боту не потрібні.

## Змінні середовища

Скопіюй `.env.example` у `.env`:

```env
DISCORD_TOKEN=токен_бота
CLIENT_ID=id_discord_application
GUILD_ID=id_тестового_сервера
DEFAULT_VOLUME=50
MAX_QUEUE_SIZE=100
```

- `DISCORD_TOKEN` — секретний token бота.
- `CLIENT_ID` — ID Discord Application.
- `GUILD_ID` — необов’язковий ID тестового сервера. З ним команди реєструються майже одразу.
- `DEFAULT_VOLUME` — початкова гучність від `0` до `100`.
- `MAX_QUEUE_SIZE` — максимальна кількість треків, що очікують у черзі.

Не додавай `.env` до git і не передавай token у Dockerfile.

За нестандартного локального встановлення можна додатково задати `YT_DLP_PATH` та `FFMPEG_PATH`.

## Запуск локально

```bash
npm install
cp .env.example .env
npm run deploy:commands
npm start
```

Режим розробки з автоматичним перезапуском:

```bash
npm run dev
```

Тести:

```bash
npm test
```

## Запуск через Docker

Створи `.env`, після чого зареєструй команди:

```bash
docker compose run --rm discord-music-bot npm run deploy:commands
```

Запусти бота:

```bash
docker compose up -d --build
```

Логи:

```bash
docker compose logs -f
```

Зупинка:

```bash
docker compose down
```

Образ побудований на `node:22-slim`; `ffmpeg` і актуальний `yt-dlp` встановлюються під час build.

## Реєстрація slash-команд

```bash
npm run deploy:commands
```

Якщо `GUILD_ID` заповнений, команди реєструються лише на вказаному сервері й з’являються швидко. Якщо `GUILD_ID` порожній, команди реєструються глобально; їх поширення Discord може тривати довше.

Після зміни описів або параметрів команд виконай deploy повторно.

Команди публікуються глобально лише для контексту **Guild Install**, тому ними можуть користуватися учасники всіх серверів, куди встановлено бота.

Якщо команди бачить лише адміністратор або власник сервера:

1. Відкрий **Server Settings → Integrations → Echo District → Manage**.
2. Для `@everyone` увімкни використання команд і потрібних каналів.
3. Перевір, що роль користувача має **Use Application Commands** і доступ до текстового каналу.
4. Якщо бот був установлений лише для твого користувача, запроси його повторно через `npm run invite`.

## Список команд

- `/play url:<youtube_url>` — відтворити трек або додати його до черги.
- `/volume value:<0-100>` — змінити поточну та наступну гучність.
- `/queue` — показати поточний трек і до 10 наступних.
- `/clear` — очистити чергу, не перериваючи поточний трек.
- `/start` — запустити чергу або продовжити відтворення.
- `/stop` — зупинити трек, очистити чергу й відключити бота.
- `/skip` — перейти до наступного треку.
- `/pause` — поставити поточний трек на паузу.
- `/resume` — продовжити трек після паузи.
- `/nowplaying` — показати поточний трек.

Після запуску першого треку повідомлення `/play` перетворюється на панель керування. Кнопка **Додати трек** відкриває форму для YouTube URL, а прогрес оновлюється кожні 10 секунд. Керувати панеллю можуть лише користувачі, які перебувають у тому самому голосовому каналі, що й бот.

Після додавання нового треку стара панель видаляється, а оновлена надсилається останнім повідомленням у чаті. Підтвердження `/play` і форми додавання бачить лише користувач, який додав трек.

## Типові помилки

**Бот не бачить slash-команди**

Повтори `npm run deploy:commands`, перевір `CLIENT_ID` і `GUILD_ID`, а також scope `applications.commands`.

**Бот не підключається до voice channel**

Перевір дозволи View Channel, Connect і Speak для ролі бота.

**`yt-dlp` або `ffmpeg` не знайдено**

Встанови обидві програми в `PATH` або запускай проєкт через Docker. Для нестандартного шляху задай `YT_DLP_PATH` чи `FFMPEG_PATH`.

**YouTube перестав віддавати аудіо**

Онови `yt-dlp`. Для Docker перебудуй образ без кешу:

```bash
docker compose build --no-cache
docker compose up -d
```

**Відео приватне, недоступне або не має аудіо**

Бот поверне зрозуміле повідомлення й не завершить роботу аварійно. Відео, які потребують cookies або авторизації, ця базова версія не підтримує.
